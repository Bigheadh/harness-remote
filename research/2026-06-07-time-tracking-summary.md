# Phase 53 Research: Time Tracking Summary

## 研究背景
参考 Toggl Track、Clockify、Harvest 等时间跟踪工具，以及 Plane、Linear 等任务管理平台的时间统计功能。

## 参考项目
- **Toggl Track**: 提供总时间、按项目/按用户/按标签的时间分解，以及每日/每周报告
- **Clockify**: 按用户的时间报告、项目时间汇总、导出功能
- **Harvest**: 发票时间跟踪、项目预算跟踪、团队时间分析

## 可借鉴特性
1. **聚合统计**: 总时间、平均时间、按用户分解 (参考 Toggl)
2. **按优先级分解**: 了解哪种优先级的任务耗时最多 (参考 Linear 的时间分析)
3. **活跃计时器计数**: 了解当前有多少正在进行的计时
4. **7 天趋势**: 最近一周的时间投入趋势

## 实现决策
- 选择在 stats 路由中添加端点，复用现有的 TTL 缓存机制
- Store 层使用 SQL JOIN 查询优先级，避免多次查询
- Dashboard 可视化使用现有的 bar-chart 和 metric-card CSS 组件，保持视觉一致性

## 下一步研究方向
- 按日期范围过滤的时间跟踪报告
- 时间跟踪与任务完成率的关联分析
- 时间预算/目标跟踪 (如每周目标 40 小时)
